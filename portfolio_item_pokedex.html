A Pokemon Tracking Pokedex

Created: July 20, 2022

The Pokedex is another fun project I took on, that combined one of my passtime pleasures with my desire for development.

It uses a dataset that I found on Kaggle in combination with PokeAPI for the Sprites of various Pokemons. More information about both the dataset, and API can be found below.

The Main Page displays all currently added in Pokemon, with a search nagivation. At the start, the Pokedex will contain all generation 1 Pokemon automatically stored, and all additional Pokemon must be added by the user.

The search bar is made from scratch making use of React useState, in which a pokemon can be looked up, and update live on the screen.

For instance, typing "cha" will return all pokemon who have the substring of "cha" within their name.

Each one of these cards are creating via REACT components, and mapped using a data object. This allows for the user to click on a individual card, to see more information regarding any pokemon.

For the below image, I selected Pokemon #6, Charizard

The sprites are obtained automatically from PokeAPI. The data is entered from the user, from the Add Pokemon Menu. For Charizard specifically, it is generation 1, which means it was included in the starter dataset.

The following is the Add Pokemon Menu Available in the top right of the navigation bar.

I will use it to add Pokemon #250, Ho-Oh.

This will now insert the Pokemon into the correct position in the pokedex, ordered by number.

It can also be observed that the Pokedex application grabbed all the sprites for Ho-Oh from the PokeAPI, when viewing the pokemon in detail. Likewise, all the data entered by the user can also be displayed is also displayed in the statistics section.

All changes made to the application are saved in local storage, to maintain any additional pokemon added. This was done so on purpose as all pokemon data is public information, so there is not risk of classified information being leaked.

These user can at any point click the reset pokedex button, which will revert to prior any pokemon being added. This means it will revert back to generation 1 pokemon.

What I Learned: This project had a lot of expectations to live up to, as I had a lot envisioned for its appearance in particular. It needed to match the theme of Pokémon, while still being functionally sound. This required following various UX techniques to make sure I do not sacrifice usability just for the sake of having a Pokémon inspired theme.

I feel this project found an excellent balance between functionality and appearance. It required extensive time honing my CSS skills, to create a finished project that does not have components that clash with one another.
