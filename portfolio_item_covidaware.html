The Covid Case Tracking Website COVIDaware

Created: January 13, 2021

COVIDaware is a project that was created alongside 13 of my peers. This project's goal was to display COVID-19 data through various visualizations. The home page looked like the following.

It served as the foundation blocks for my interest in learning more about web-development. It was the first major project I did with a significant group. Many of us had no prior experience so it served as a great opportunity to learn. The website was designed with a large focus on functionality. This is why the website was mostly grayscale, to appeal to users who need vision assistance.

The website also adopted many industry standard UX Techniques such as the navigation bar, the search bar, and even drop-down menu's for items in the navigation bar.

This project also was a lesson on operating with various API's. For the project a google maps api, along with a twitter api was used for various components. This taught me how to adjust to different constraints of various API's. Furthermore the significance of using API's, and how valuable they are in development.

The following is one of many visualizations that were created using the google maps API:

This project also served as reinforcing my knowledge on REACT and bootstrap, which were instrumental in the creating of the project. React helped in the creation project by helping build the user interfaces. Furthermore bootstrap was a complete game changed in terms of helping with CSS. This portfolio website was created without any libraries like bootstrap, thus making the CSS much more tedious. This was a self imposed constraint to better learn CSS.

For more information regarding the project, and functionality, the user manual created by myself for the purpose of the project can be used to get a deeper understanding of the project.

COVIDaware User Manual

The website pulled data from government official websites, was converted for use in the back-end. The front-end would assign various variables to parameters, and incorporate for use in API's. This was done in inefficient manner, and needs to be re-proccessed.

Currently the source in which the data came from has changed, thus breaking the core functionality of the website. Thus it is currently decommissioned, and I am in the proccess of fixing several aspects before redeployment. A new user-interface, more efficient use of data, integrating a mysql database for back-end, and re-integrating the data source for data are all tasks that must be completed before redeployment.
